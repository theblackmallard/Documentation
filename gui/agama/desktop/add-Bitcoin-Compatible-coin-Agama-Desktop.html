# Add a Bitcoin Compatible coin to Agama Desktop

This guide will help you to add Bitcoin compatible coins into Agama desktop wallet. You have to edit the backend part and the UI part to add. Follow the guide carefully.

# Backend

  • Add network params pbca26/agama-wallet-lib:src/bitcoinjs-networks.js@dev. Make use of isPoS or isZcash flags if applicable in your case.

  • Add an electrum server pbca26/agama-wallet-lib:src/electrum-servers.js@dev#L1

  • Add a safe fixed fee (per transaction) pbca26/agama-wallet-lib:src/fees.js@dev#L1.

  • Add an explorer pbca26/agama-wallet-lib:src/coin-helpers.js@dev#L62.

  • Submit a PR

# UI

  • Drop a 100 x 100 px (better 200 x 200 px) logo into KomodoPlatform/EasyDEX-GUI:react/src/assets/images/cryptologo/btc@dev .

  • Add your coin name to EN translation file KomodoPlatform/EasyDEX-GUI:react/src/translate/coins.js@dev, look for CRYPTO.

  • Add your coin to coins helper file KomodoPlatform/EasyDEX-GUI:react/src/util/coinHelper.js@dev#L300.

  • Submit a PR to dev branch of each repo .

# How to get network parameters

  • pubKeyHash: KomodoPlatform/komodo:src/chainparams.cpp@fbb3b3e#L169

  • scriptHash: KomodoPlatform/komodo:src/chainparams.cpp@fbb3b3e#L170

  • wif: KomodoPlatform/komodo:src/chainparams.cpp@fbb3b3e#L171

  • bip32 public: KomodoPlatform/komodo:src/chainparams.cpp@fbb3b3e#L172

  • bip32 private: KomodoPlatform/komodo:src/chainparams.cpp@fbb3b3e#L173

If you can't find chainparams.cpp in your code base, all of the information and parameters required are normally contained within but not limited to these files (depends on your coin):

  • init.cpp: https://github.com/litecoin-project/litecoin/blob/master/src/init.cpp
  • base58.h: https://github.com/litecoin-project/litecoin/blob/master/src/base58.h
  • chainparamsbase.h: https://github.com/litecoin-project/litecoin/blob/master/src/chainparamsbase.h

Please make sure the BTC compatible coin is working in Agama before making a PR. Pull requests containing partial information or not working assets/servers will remain unmerged until all requirements are fulfilled.
